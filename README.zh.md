# My Ideals - 记录你的生写真收藏

[English](./README.en.md) | [日本語](./README.ja.md) | [中文](./README.zh.md)

My Ideals 是一个用于管理偶像生写真收藏的 Web 应用。

- ✅ 标记已拥有的照片
- 📊 一览收藏进度
- 🔍 快速搜索和筛选
- 💾 数据保存在浏览器中 - 无需注册账号

## 快速上手

### ⚠️ 警告

本应用所有数据仅保存在浏览器本地，清空浏览器数据、使用无痕模式或更换设备都将导致数据永久丢失，请务必定期导出备份数据。**开发者不对因数据丢失造成的任何损失负责，也无法协助恢复已丢失的数据。**

https://my-ideals.notequal.me/

### 创建档案

1. 从以下菜单中选择 **“新建档案”**。
   - **电脑端**：点击右上角的下拉列表
   - **手机端**：点击右上角的三横线（汉堡菜单）
2. 粘贴模板 URL（从 [此处](https://github.com/monaka-ikonoi/my-ideals/issues/1) 或模板制作者处获取）
3. 输入档案名称
4. 点击 **“创建”**

### 筛选成员

- 点击成员筛选按钮只显示特定成员
- 适用于包含多个成员的模板

### 记录收藏

1. 浏览各个收藏集
2. 点击任意条目标记为 **已拥有** ✓
3. 再次点击取消标记
4. 进度自动保存

### 备份数据

- 点击右上角的 **"导出"** 下载数据文件
- 妥善保管，需要时可以导入恢复

## 工作原理

**模板 (Template)** = 社区创建的照片列表

模板定义了一个系列有哪些照片（例如："2024夏季演唱会周边"）。模板作者会维护这些列表，你不需要手动输入每个条目。只需粘贴模板链接即可开始使用。

**档案 (Profile)** = 你的个人收藏记录

档案基于模板，记录你拥有哪些照片。你可以创建多个档案（例如：按团体分类、交换用等）。

```
模板（共享）         档案（个人）
┌─────────┐       ┌─────────┐
│ A       │       │ A    ✓  │  ← 已有
│ B       │  -->  │ B       │
│ C       │       │ C    ✓  │  ← 已有
│ ...     │       │ ...     │
└─────────┘       └─────────┘
```

## 常见问题

**Q：数据保存在哪里？**

A：所有数据保存在浏览器的本地存储中，不会发送到任何服务器。

**Q：如果清除浏览器数据会怎样？**

A：收藏数据会丢失。清除前请务必导出备份。

**Q：可以在多个设备上使用吗？**

A：可以，但数据不会自动同步。请从一台设备导出，然后在另一台设备导入。

**Q：什么是模板？**

A：模板是定义照片列表的 JSON 文件。模板作者维护这些列表，用户无需手动输入每个条目。

**Q：模板链接失效了怎么办？**

A：点击 **"编辑模板链接"** 更新链接。如果模板已迁移，请向模板作者获取新链接。
